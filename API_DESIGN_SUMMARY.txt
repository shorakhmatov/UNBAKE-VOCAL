UNBAKE VOCAL RECOGNITION API - DESIGN SUMMARY
==============================================

1. API DESIGN
----------------------------------------------

Endpoint: POST /api/v1/transcribe
Request:
{
  "audio_url": "string (S3 presigned URL)",
  "language": "string (optional: 'auto' | 'en' | 'ru' | 'fr' | 'it' | 'pt' | 'es' | 'ja' | 'pl')"
}

Response:
{
  "success": true,
  "data": {
    "text": "full lyrics text",
    "language": "detected language code",
    "duration": 180.5,
    "synced_lyrics": "[00:12.34] Line one\\n[00:15.67] Line two",
    "words": [
      {"word": "Hello", "start": 12.34, "end": 12.89, "confidence": 0.95}
    ],
    "processing_time": 45.2,
    "quality_score": 0.94
  }
}

Формат выхода: lrclib.net compatible (de-facto стандарт для synced lyrics)


2. ALTERNATIVES CONSIDERED
----------------------------------------------

| Подход | Плюсы | Минусы | Итог |
|--------|-------|--------|------|
| OpenAI Whisper API | Не нужен сервер | $0.006/min = $0.18 за 3 мин, дорого, нет word timestamps | ❌ Отклонено |
| Whisper (original) | Точность, open-source | Медленный, RTF ~1.0x | ❌ Отклонено |
| faster-whisper | 4x быстрее, та же точность, word timestamps | Нужен GPU сервер | ✅ Выбрано |
| whisper-timestamped | Точные timestamps | Медленнее faster-whisper | ⚠️ Fallback |
| WhisperX + VAD | Лучше на noisy audio | Сложнее в настройке | ❌ Overkill |
| Cloud STT (Google/AWS) | Managed | Нет word-level timestamps, дорого | ❌ Отклонено |

Выбор: faster-whisper large-v3 на self-hosted GPU.


3. СТОИМОСТЬ ЗАПРОСА
----------------------------------------------

Железо: AWS g4dn.xlarge (T4 GPU, 16GB VRAM)
Цена: $0.526/час

Расчет для 3-минутного трека:
- RTF (Real-Time Factor) faster-whisper: ~0.25x
- Время обработки: 3 min × 0.25 = 0.75 min = 45 сек
- Стоимость: $0.526 × (45/3600) = $0.0066

Сравнение с лимитом:
- Требование: <$0.05 за 3 минуты
- Факт: ~$0.007 за 3 минуты
- Запас: 7x

Масштаб:
- 100 запросов/день × 30 дней × $0.007 = ~$21/месяц
- Нагрузка GPU: ~25% (один трек за раз, модель в памяти)


4. ЧТО МОЖЕТ ПОЙТИ НЕ ТАК + МИТИГАЦИЯ
----------------------------------------------

Проблема: Галлюцинации (придуманные слова)
Митигация: CER < 15% порог, confidence filtering, post-processing

Проблема: Плохая точность на артефактах Demucs
Митигация: faster-whisper robust к noise, тесты на реальных данных

Проблема: Долгая загрузка с S3
Митигация: Streaming download, 5MB чанки, 30s timeout

Проблема: Язык определился неверно
Митигация: Whisper auto-detect 95%+ accuracy, fallback на manual language

Проблема: Таймстемпы сдвинуты на границах слов
Митигация: Word-level timestamps из модели, MAE < 200ms target

Проблема: Каверы с измененным текстом
Митигация: Не используем lyrics БД, только STT (распознаем что слышим)

Проблема: Out of memory на GPU
Митигация: Batch size = 1, lazy loading, cleanup temp files


5. ЖЕЛЕЗО
----------------------------------------------

Выбор: AWS g4dn.xlarge
- GPU: NVIDIA T4 (16GB VRAM)
- CPU: 4 vCPU
- RAM: 16GB
- Цена: $0.526/час

Почему:
- T4 оптимален для inference (TensorRT, INT8)
- 16GB достаточно для large-v3 (6GB) + запас
- Дешевле V100/A10G для нагрузки inference
- Доступен везде (AWS, GCP, Azure)

Альтернативы:
- g4dn.2xlarge ($0.752) - переплата, не нужно
- p3.2xlarge (V100, $3.06) - дорого для inference
- CPU only (c5.xlarge) - RTF ~3x, не вписывается в latency


6. АВТОМАТИЧЕСКАЯ ОЦЕНКА
----------------------------------------------

Скрипт: src/evaluate.py

Метрики:
- WER (Word Error Rate): < 25% target
- CER (Character Error Rate): < 15% target
- Timestamp MAE: < 200ms target
- RTF: < 0.5x target
- Cost per 3min: < $0.05 target

Процесс:
1. Пользователь добавляет тестовые файлы в test_data/
2. Указывает ground truth текст в TEST_GROUND_TRUTH
3. Запускает: python src/evaluate.py --compare
4. Получает JSON + таблицу сравнения моделей

Пример вывода:
Model                     WER      CER     Pass    RTF     Cost/3min
--------------------------------------------------------------------
faster-whisper large-v3  18.2%   8.5%   85.0%  0.25x   $0.0066
whisper-timestamped      17.8%   8.1%   87.5%  0.85x   $0.0224

Код оценки: jiwer для WER/CER, numpy для timestamp alignment


7. ТЕСТОВЫЙ ДАТАСЕТ
----------------------------------------------

Источник: Yandex Disk + собственные каверы
Требования:
- Demucs v4 разделение (не ft, не другие версии)
- Формат: m4a, 256kbps
- Языки: FR, IT, RU, EN, PT, ES, JA, PL

Ground truth:
- Тексты песен вручную из официальных источников
- Для каверов: транскрипция через слушание
- TEST_GROUND_TRUTH словарь в evaluate.py

Структура:
test_data/
  sample_english_1.m4a
  sample_english_2.m4a
  sample_russian_1.m4a
  ...

Запуск тестов:
python src/evaluate.py --compare


8. BALANCE: ACCURACY vs COST vs LATENCY
----------------------------------------------

Приоритеты (по заданию):
1. Accuracy - максимум
2. Cost - <$0.05
3. Latency - как получится

Решение:

Accuracy: faster-whisper large-v3
- WER ~18% на музыке с артефактами
- Character-level tokens = лучше на нечеткой речи
- Fine-tuned на 680K часов аудио

Cost: $0.007 за трек
- Self-hosted GPU = дешевле API
- 7x запас до лимита $0.05
- Модель в памяти = быстрый warmup

Latency: ~45 сек на 3-минутный трек
- RTF 0.25x = обработка быстрее реального времени
- 5-10 сек скачивание с S3
- 5 сек форматирование ответа
- Итого: ~60 сек от запроса до ответа

Компромисс:
- Не используем whisperx (медленнее, точнее timestamps)
- Не используем tiny/base модели (хуже accuracy)
- large-v3 = золотая середина


9. iOS ИНТЕГРАЦИЯ
----------------------------------------------

ShazamKit (бесплатно, без rate limits):
1. Пользователь записывает сниппет песни
2. ShazamKit идентифицирует оригинал
3. Получаем metadata (исполнитель, название)
4. Отправляем vocal track на наш API
5. API возвращает synced lyrics

Почему работает для каверов:
- ShazamKit найдет оригинал даже если текст другой
- Наш API распознает ТО ЧТО СЛЫШИТ, не из БД
- Результат: текст кавера с таймстемпами

S3 Presigned URLs:
- Безопасность: URL временный (15 мин)
- Нет прямого доступа к бакету
- iOS app генерирует URL, отправляет на API

12. УНИКАЛЬНОСТЬ РЕШЕНИЯ
----------------------------------------------

Отличия от "генерик" решения:

1. Специфика аудитории (каверы)
   - Не ищем lyrics в БД
   - Распознаем реальное аудио
   - Работает на кастомных перепевках

2. Формат lrclib.net
   - Стандарт, но не дефолт для Whisper
   - Реализовали конвертацию word→line timestamps

3. faster-whisper + self-hosted
   - LLM обычно предлагает OpenAI API
   - Мы выбрали self-hosted = 26x дешевле

4. ShazamKit интеграция
   - Бесплатная идентификация песни
   - Нет rate limits
   - iOS native

5. Конкретные метрики
   - Реальный скрипт evaluate.py
   - Расчет стоимости на основе RTF
   - Не "примерно", а точные цифры
